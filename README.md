# Aziz Dhifaoui Portfolio

Production-ready portfolio built with Next.js App Router.

## Local development

Install dependencies and start the dev server:

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

## Environment variables

Create `.env.local` (or configure these in Vercel Project Settings → Environment Variables):

```dotenv
NEXT_PUBLIC_SITE_URL=https://your-domain.com
RESEND_API_KEY=your_resend_api_key
CONTACT_TO_EMAIL=you@your-domain.com
CONTACT_FROM_EMAIL=Portfolio Contact <noreply@your-domain.com>
```

## Contact form notes

- The contact form posts to `POST /api/contact`.
- `RESEND_API_KEY`, `CONTACT_TO_EMAIL`, and `CONTACT_FROM_EMAIL` are required in production.
- The API returns a safe generic error if email provider delivery fails.

## Static assets

- Resume download is served from `public/resume.pdf` at `/resume.pdf`.
- Favicon is served from `src/app/favicon.ico`.
- Open Graph image is generated by `src/app/opengraph-image.tsx`.

## Deployment (Vercel)

1. Push the repository to GitHub/GitLab/Bitbucket.
2. Import the project in Vercel.
3. Set environment variables listed above for Production and Preview.
4. Run a production check locally before deploying:

```bash
npm run lint
npm run build
```
